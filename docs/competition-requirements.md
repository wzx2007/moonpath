# MoonBit 开源生态项目贡献赛要求摘要

来源：GitLink 赛题页 `track1_2026MoonBit`，抓取时间：2026-05-25。

## 赛事基本信息

- 赛题名称：MoonBit 开源生态项目贡献赛。
- 赛事时间：2026-03-20 至 2026-07-11。
- 报名/提交截止：2026-07-11 00:00。
- 主题：围绕 MoonBit 开源生态建设，产出有真实场景和可复用价值的开源库、工具或组件。

## 作品方向

参赛项目可以原创实现，也可以移植或重写成熟语言生态中的优秀开源库。推荐方向包括：

- 基础数据结构与算法：indexmap、bitmask、Graph 算法库、Pathfinding 工具库等。
- 工程基础设施与工具链：日志/tracing、构建工具、模板渲染、benchmark 工具等。
- 系统能力与运行时框架：deterministic simulation、寄存器分配、protobuf 类序列化、actor 框架等。
- 应用生态：Markdown to HTML、图表数据生成、游戏引擎绑定、数据库绑定等。

本项目选择方向：基础数据结构与算法 / Graph 与 Pathfinding 工具库。

## 申报材料

申报材料建议控制在一页 PDF，内容包括：

- 项目名称。
- 项目简介。
- 项目方向与适用场景。
- 拟实现的核心功能。
- 是否原创、移植或参考已有项目。
- 如为移植项目，说明参考项目名称、来源链接和许可证。
- GitHub 仓库链接，含 10-20 个有效 commits。
- GitLink 仓库链接，需与 GitHub 同步。

## 验收要求

- 项目以 MoonBit 为主要实现语言。
- GitHub 和 GitLink 仓库公开可访问，提交记录清晰。
- 源代码结构清晰，能够完成声明的核心功能。
- README 说明项目目标、安装方式、使用方法和示例，并可复现。
- 使用持续集成工具覆盖检查、构建、测试流程。

## 仍需参赛者本人完成

- 使用本人 CCF/GitHub/GitLink 账号报名。
- 将本地仓库推送到公开 GitHub 仓库。
- 在 GitLink 创建/同步公开仓库。
- 将 `docs/project-proposal.md` 转为 PDF 并填写个人信息与仓库链接。
- 以真实开发过程补足 10-20 次有效提交，避免空提交或机械拆分。
